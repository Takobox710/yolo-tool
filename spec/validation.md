# 模型验证

验证页左右比例当前为 `1:3`，左侧保持紧凑，右侧图片区和详情表纵向占位比例为 `2:1`：

- 左侧：模型配置（含下拉框自动扫描模型）、置信度/IoU/图片尺寸、检测模式与输入源、检测按钮区、日志文本框（不显示"检测日志"标题）；普通检测模式下日志文本框应占用左侧面板的剩余高度并延伸到底部。
- 右侧：源图、检测结果图、检测结果详情表。

支持输入源：

- 图片检测
- 视频检测
- 摄像头检测
- 数据集验证

支持自动扫描 `result/**/weights/*.pt`，也可手动选择模型。

- 模型验证页的“选择模型”下拉框默认只显示各训练目录下的 `best.pt`。
- 当系统设置开启“模型验证显示 last”时，模型验证页的“选择模型”下拉框才额外显示各训练目录下的 `last.pt`。
- 若当前已选中某个训练目录的 `last.pt`，随后关闭“模型验证显示 last”，模型验证页应自动回退到同一训练目录下的 `best.pt`。

验证页 UI 注意事项：

- 左侧和右侧采用可缩放的 `1:3` 权重，不要用固定像素宽度锁死，否则小窗口会截断右侧内容。
- 下拉控件使用 Qt 原生 `QComboBox` 并保持与普通输入框一致的边框，不要再用额外白底外壳模拟边框。
- 所有面板顶部边框和圆角必须完整显示，标题栏不能覆盖面板上边框，也不能让顶部圆角变直角。
- 选择模型使用上方标签、下方可编辑下拉框，右侧带"选择"按钮，可手动选择 `.pt` 模型。
- 验证页“图片尺寸”放在 `IoU` 后面，使用可编辑下拉框，内置 `640`、`960`、`1280` 三个候选值，同时允许手动输入其他尺寸；该值必须实际传递给检测推理的 `imgsz` 参数。
- 不显示"检测源配置"标题；模型路径、检测模式、输入源等左侧控件必须有统一内边距，不能贴面板边缘。
- 验证页外层保持标准 16px 页面内边距；右侧内部装配布局不额外叠加默认 margin，避免右侧模块与软件边缘的距离偏大或贴边。
- 置信度与 IoU 放在同一行；置信度输入框左侧要与选择模型输入框左侧对齐，IoU 与右侧输入框距离要自然。
- 源图和检测结果图与面板边框之间要保留内边距，不能贴边。
- 源视频和检测后视频预览面板使用等权横向宽度，两个视频区域保持一致。
- 图片检测与视频检测相互切换时应一次性完成工具栏、图片预览和视频预览控件的显示状态更新，切换过程中不得出现中间画面闪动。
- 图片检测源图和检测结果图上方要有批量检测结果工具栏，按钮顺序为：上一张、下一张、第一张、最后一张、列表、打开保存文件夹，并显示计数；视频检测改为视频播放工具栏。
- 检测结果详情表不显示"序号"列，只显示类别、置信度、坐标、尺寸、角度。
- 批量检测时保持右侧显示第一张图片，不随进度切换，后台持续检测，前端维持在第一张；检测前上一张、下一张、第一张、最后一张切换输入源预览，检测开始后这些按钮切换已检测完成的结果，不得触发重新检测。
- 检测开始前，图片输入源加载完成后源图区域立即显示第一张图片；输入源列表可选择并预览任意图片。单张图片或拖入图片直接预览对应图片，拖入视频或选择单个视频时源视频暂停显示第一帧。切换检测模式或输入源时清除旧的图片、视频和结果显示，检测开始后恢复原有结果列表和缓存结果逻辑。
- 检测前图片工具栏右侧计数显示当前源图片位置与输入源总数，例如 `1/20`；上一张、下一张、第一张、最后一张或列表切换预览时同步更新。检测开始后计数恢复为检测结果数量逻辑。
- 选择"图片检测"或"视频检测"时，主按钮统一显示"开始检测"；图片检测只处理图片，视频检测只处理视频。
- 选择"摄像头检测"时，右侧结果区改为实时预览语义，不进入批量结果列表；隐藏批量结果工具栏，上一张、下一张、第一张、最后一张和列表按钮都不应用于摄像头历史帧切换，计数区域显示"实时预览"。
- 选择"数据集验证"时，主按钮统一显示"开始检测"，左侧显示 `数据集 YAML`、`选择验证源`、`输出文件夹` 和 `打开保存目录`；`选择验证源` 带文件夹选择按钮，支持自定义验证目录；右侧图片区/结果表隐藏，仅显示"验证日志"。
- 数据集验证模式下左侧配置项按内容高度从顶部紧凑排列，不因右侧验证日志面板的高度被均匀拉开；普通检测模式的左侧日志框仍占用剩余高度。
- 摄像头检测日志需持续显示实时帧率（FPS）。
- 摄像头检测在无检测目标时，检测结果图仍必须继续显示当前摄像头画面，只是不叠加检测框，严禁出现黑屏或空白结果图。
- 批量检测顺序必须使用自然数字排序，例如 `1.jpg, 2.jpg, 3.jpg, 10.jpg, 100.jpg`。
- 点击"列表"应直接读取当前输入源文件夹，列表只显示文件名，不显示序号；列表窗口初始尺寸为 `320 x 520`，最小尺寸为 `200 x 200`，底部带"搜索文件名"输入框。
- 列表选择已检测完成的图片或视频时，应立即跳转显示对应缓存结果；选择未检测文件只更新当前选择，不自动触发检测。
- 图片检测/视频检测/摄像头检测应通过隐藏子进程执行，任务结束后由子进程退出回收主要 `torch` / `ultralytics` 推理内存；检测完成或停止后不得关闭 GUI 主窗口。
- 批量检测结果缓存只保留结果路径与轻量元数据；需要回看某张结果时再从磁盘重载图片，避免一次检测数百张图片后 GUI 内存持续堆积。
- 检测和训练均只允许一次启动，按钮在运行期间禁用，任务结束后恢复。

检测源配置约定：

- 检测模式是下拉框，包含"图片检测"、"视频检测"、"摄像头检测"和"数据集验证"，默认仍为"图片检测"。
- 选择"图片检测"时，`输入源` 下拉框提供 `全部图片`、`训练图片`、`验证图片`、`测试图片`，并在最下方增加 `单张图片`；选择按钮默认使用 Windows 原生文件夹对话框，只有当前输入源为 `单张图片` 时使用图片文件对话框。
- 选择"图片检测"时，若输入源选择 `全部图片`，读取软件设置中的图片路径（默认项目根目录 `images/`）；若选择 `训练图片`、`验证图片`、`测试图片`，分别读取 `data/train/images`、`data/val/images`、`data/test/images`；选择 `单张图片` 后只收集所选图片文件。
- 选择"视频检测"时，`输入源` 下拉框提供 `批量视频`、`单个视频`；选择 `批量视频` 时使用 Windows 原生文件夹对话框，选择 `单个视频` 时使用视频文件对话框；该模式显示视频播放进度条和左侧检测控制按钮。
- 使用“输入源”右侧选择按钮选取自定义文件夹后，界面显示相对于项目根目录的路径；项目外路径使用 `..` 表示，实际检测时仍解析为绝对路径。单个图片或视频通过拖入验证页加载。
- “视频检测”模式自动进入视频检测界面：隐藏上一张、下一张、第一张、最后一张等图片结果切换控件，打开页面时显示当前第一个视频的第一帧但不自动播放；顶部显示可拖动的视频播放进度条，左侧为独立的三角播放/暂停按钮，右侧保留“上个视频”“下个视频”“列表”和“打开保存文件夹”。
- 视频播放进度条跟随当前预览视频播放位置，拖动时同步定位源视频和检测后视频；检测过程中的百分比和上一秒速度只写入日志，不覆盖播放进度条。
- 视频自然播放结束后，播放按钮应自动恢复为三角播放图标并取消选中状态，播放进度条停留在末尾。
- 模型验证页面支持拖入单个图片或视频文件；拖入后按文件类型自动切换检测模式，输入源显示相对路径，点击检测按钮只检测该文件。
- 批量检测切换到第二、第三个视频时，右侧继续保持当前选中的预览视频；只有点击“上个视频”“下个视频”或列表选择后才切换预览。
- 视频检测日志每秒最多追加一条，显示当前视频完成百分比、当前帧数和上一秒实际检测帧数；视频检测完成后提示“检测完成，请开启下一个视频检测”。
- 视频检测结果写出为 MP4 视频，不保存视频帧对应的 YOLO 标注 TXT 文件；视频帧结果不进入图片结果导航缓存。
- 选择"摄像头检测"时用摄像头下拉框替换输入源。
- 选择"数据集验证"时显示 `数据集 YAML` 与 `选择验证源`，可通过按钮选择自定义文件夹；点击"开始检测"时仅临时改写 `data.yaml` 中的 `val:` 指向；待验证进程退出后必须恢复原文件。
- 检测模式右侧不要放路径选择图标。
- 不显示"检测控制"标题。
- 四种检测模式的主按钮统一显示"开始检测"，并保留"停止"按钮；点击"停止"后应请求结束当前检测子进程，并在子进程真正退出后统一恢复按钮状态。
