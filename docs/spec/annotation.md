# 数据标注规格

## 右侧栏

- 右侧栏使用“任务类别”选择 `detect` 或 `obb`，使用“目标类型”选择当前类别。
- 未选中标注时，“目标类型”表示新建标注默认使用的类别。
- 选中画布中的标注或右侧标注列表项后，“目标类型”立即切换到该标注的类别。
- 选中标注时修改“目标类型”，会立即修改该标注的类别并刷新标注列表；未选中标注时只修改后续新建标注的默认类别。
- 类别下拉框切换后画布立即重绘；标注轮廓、标签和控制点使用同一类别颜色。
- 类别名称保存于当前项目的 `data/runtime/settings.json`。切换项目后重新读取项目类别，不沿用上一个项目的类别。
- 进入标注页或扫描项目图片时，会读取项目 Labelme 标注目录下所有 JSON 的非空 `label`；设置文件中不存在的类别会按扫描顺序追加并保存。

## 画布编辑

- 编辑模式下，有向矩形（包括 OBB、镜像有向矩形和直线扩展生成的旋转矩形）在四条边的中心各显示一个圆形旋转控制点。
- 选择并拖动边中点控制点时，标注围绕四个角点的几何中心旋转；旋转不改变矩形中心、长宽和四角点顺序。
- 有向矩形的四个角点仍用于调整矩形尺寸和方向；整体拖动标注时，旋转控制点与角点同步移动。旋转控制点只在编辑模式下可命中和拖动，绘制预览仅显示控制点。
- 镜像有向矩形默认使用普通有向矩形控制点；开启“优化镜像有向矩形编辑”后，编辑态显示中心线，并仅显示中心线两端和两条长边中心四个控制点，不显示四角控制点。拖动中心线端点按中心线端点逻辑调整，拖动任一长边中心点时，另一侧按中心线对称同步扩展或缩小。
- Labelme 文件中的镜像有向矩形和直线扩展均保持 `shape_type: oriented_rectangle`，并通过 `flags.yolo_tool_shape` 保存 YOLOTool 内部形状；不带该标记的外部 `oriented_rectangle` 仍按普通有向矩形加载。

## 类别管理

- “管理类别”支持新增、重命名和删除类别。
- 类别管理窗口底部按钮显示为“确定”和“取消”。
- 重命名类别保持类别索引不变，已有标注继续引用原类别。
- 删除类别前会统计当前项目标注中依赖该类别的数量；数量不为零时阻止删除，并提示依赖数量。
- 类别管理窗口底部左侧提供“转换类别”按钮，点击后打开独立设置窗口；窗口中选择源类别和目标类别，点击“确定”后将所有当前项目标注的源类别改为目标类别，“取消”则不执行转换。
