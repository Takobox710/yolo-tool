# 系统设置

系统设置页面显示 Python、Torch、Ultralytics、PySide6、OpenCV、Pillow、psutil 和程序版本。系统信息区域采用白色外框 + 灰色内卡样式（仿照参考图片），2x4 网格排列，每 0.5 秒自动刷新一次。

系统设置页面还包含：

- 系统信息区域放在页面最上方。
- 第二行最后一个卡片固定显示程序版本；当前版本号为 `1.2.4`。
- 系统信息卡片优先展示环境是否就绪与关键依赖版本，不再显示 Pixi、GPU、CPU、内存、磁盘、显存占用。
- 除程序版本外，其余卡片统一显示为 `版本号：可用/不可用`。
- `Torch` 卡片不再显示 `Torch/CUDA` 合并标题；若 `torch` 可用但 CUDA 不可用，则显示为 `2.12.1+cu130：CUDA不可用` 这类格式。
- 系统设置下方文本区域用于显示程序日志，而不是环境 JSON；左上角显示标题“程序日志”。这里优先展示程序级运行信息、后台异常和通用报错，不重复承载训练页、验证页、AI 预标注、标注转换、图片压缩等页面各自已有的专属日志。
- 路径设置中，标注转换、标注预览、批量重命名和数据标注共用图片目录；数据标注、标注转换和批量重命名共用 Labelme 标注目录；标注预览和标注转换共用 YOLO 标注目录。图片压缩源目录独立保存。
- 系统信息下方使用同一行放置以下控件：`训练前显示自定义命令框`、`显示配置解释符号`、`模型验证显示 last`、`恢复默认设置`。
- "训练前显示自定义命令框"开关，控制训练页是否弹出命令编辑对话框。
- "显示配置解释符号"开关，控制是否显示字段名后的 `ⓘ`。
- "模型验证显示 last"开关，控制模型验证页“选择模型”下拉框是否显示训练结果中的 `last.pt`；默认关闭，关闭时只显示 `best.pt`。
- 系统设置页还保留 `多类别分布模式` 开关，并放在该行控件的最左侧。
- 系统设置页不再提供“启动自动加载 torch”之类的常驻预热开关；GUI 默认不因打开设置页而把 `torch` 推理运行时长期留在主进程内。
- `Torch/CUDA` 卡片信息应通过短生命周期后台子进程读取，读取完成后立即退出，避免仅查看环境信息就把推理内存抬到验证/预标注任务时的水平。
- "恢复默认设置"按钮，点击后将当前项目的设置恢复到代码默认值，但保留当前项目目录不变。

关于"显示配置解释符号"开关：

- 关闭时只隐藏 `ⓘ`，不要清空 tooltip。
- 关闭后鼠标悬停在字段名称本身，仍应能通过默认 Qt tooltip 延迟看到解释。
- 不要再实现"关闭后完全无解释"或需要额外自定义浮层的方案。
